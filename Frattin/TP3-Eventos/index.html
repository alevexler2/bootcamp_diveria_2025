GET /events

Este endpoint permite obtener una lista de eventos filtrados según criterios opcionales.

Parámetros de consulta (opcionales):

  • title (string): Filtra eventos cuyo título contenga el texto especificado.
  • from (date): Fecha desde la cual se buscan eventos (inclusive).
  • to (date): Fecha hasta la cual se buscan eventos (inclusive).

Si no se especifican parámetros, se devuelve la lista completa de eventos.
A sí mismo se pueden utilizar uno, dos o los tres parámetros para filtrar los eventos.

Presione el botón GET para realizar la búsqueda.

GET /events/{id}

Este endpoint permite obtener un único evento a través de su ID correspondiente.

Parámetro de consulta:

  • ID (number): Filtra eventos cuyo ID sea el número especificado.

El parámetro es obligatorio y debe ser un número entero positivo.

Presione el botón GET para realizar la búsqueda.

GET /events/most-reserved

Este endpoint permite obtener uno o varios eventos que tengan la mayor cantidad de reservas hechas.

Este endpoint no requiere parámetros

Presione el botón GET para realizar la búsqueda.

GET /events/available

Este endpoint devuelve todos los eventos que aún tienen cupos disponibles, es decir, aquellos que no están completamente reservados.

Este endpoint no requiere parámetros

Presione el botón GET para realizar la búsqueda.

POST /events

Este endpoint permite crear un nuevo evento.

Parámetros de creación:

  • title (string): Determina el Título del evento
  • date (date): Determina la Fecha del evento
  • capacity (number): Determina el cupo total

Todos los parámetros son obligatorios y cada uno tiene una cierta validación.

Presione el botón POST para crear un nuevo evento.

PUT /events/{id}

DELETE /events/{id}

A continuación se integran dos enpoints utilizados tanto para actualizar como para eliminar un evento existente.

Parámetros para actualizar:

  • title (string): Nuevo título del evento
  • date (date): Nueva fecha del evento
  • capacity (number): Nuevo cupo total

Todos los parámetros son obligatorios para la actualización y deben cumplir con las validaciones correspondientes.

Parámetros para eliminar:

  • id (number): ID del evento a eliminar

La eliminación se realiza enviando el ID del evento como parámetro en la URL.

Presione el botón Actualizar lista para cargar los eventos.
